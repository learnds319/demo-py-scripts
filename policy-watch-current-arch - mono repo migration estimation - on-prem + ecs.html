Hybrid Migration Command Center

Strategic Decision: Hybrid Oct 26, 2023

Executive Summary

Policy Watch will adopt a Hybrid Architecture. Critical data acquisition (Scrapers/Rules) will remain On-Premises to leverage existing network trust and legacy stability. Intelligence extraction (Extractor/Analyzer) remains Cloud-Native (ECS). Orchestration will bridge the gap via API Wrappers.

Infrastructure Mix

Target: 40% On-Prem (Wrappers) / 60% Cloud

Orchestration Model

Unified Event-Driven Control Plane

Hybrid Goals

  • 1 Retain On-Prem Scrapers (IP Trust)
  • 2 Centralize Orchestration (Step Functions)
  • 3 Wrap Legacy Scripts in APIs
  • 4 Maintain Legacy DB Connectivity

Hybrid Orchestration

Strategy: Remote Triggers via Step Functions

AWS Step Functions will act as the "Conductor," sending remote commands to the On-Prem servers (via API Gateway/VPN) to trigger local scripts, and then coordinating the Cloud-Native processing.

Architect's Critical Risks

Hybrid Connectivity

Reliability depends entirely on the VPN/Direct Connect link. Resolution: Implement redundant connections and strict timeout policies in Step Functions.

API Security

Exposing On-Prem scripts as APIs requires strict auth. Resolution: Use Mutual TLS (mTLS) or API Keys via Secrets Manager.
